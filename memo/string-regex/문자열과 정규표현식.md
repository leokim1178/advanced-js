## 정규식

1. common character

- `\d`, `[0-9]`: d는 decimal, 숫자 (0-9)
  - `[9-0]`: 불가능, 역순으로는 범위를 지정할 수 없다
- `\D`, `[^0-9]`: 숫자가 아닌 문자
- `\w`, `[a-zA-Z0-9_]`: w는 word, 단어 문자 (알파벳 대소문자, 숫자, 언더스코어)
  - `[ㄱ-ㅎㅏ-ㅣ가-힣]`: 한글 문자 포함시키려면 이렇게 추가해주면 된다
  - `[a-zA-Z0-9_\-]`: 하이픈(-)도 포함시키려면 언더스코어 뒤에 `\-`를 추가해주면 된다
- `\W`, `[^a-zA-Z0-9_]`: 단어 문자가 아닌 문자
- `\s`, `[ \t\n\r\f\v]`: s는 space, 공백 문자 (스페이스, 탭, 줄바꿈 등)
- `\S`, `[^ \t\n\r\f\v]`: 공백 문자가 아닌 문자
- `\xHH`: 16진수 2자리(HH)로 표현된 ASCII 문자 코드
  - `\x41`: 'A' (16진수 0x41 = 10진수 65)
  - `\x7A`: 'z' (16진수 0x7A = 10진수 122)

2. flags

- `g`: global, 전체 문자열에서 모든 매치를 찾음
- `i`: ignore case, 대소문자 구분 없이 매치
- `m`: multiline, 여러 줄 모드 (줄 시작과 끝을 인식)
- `s`: dotall, `.`이 줄바꿈 문자도 매치하도록 함
- `u`: unicode, 유니코드 모드 (유니코드 문자 인식)
- `y`: lastIndex, sticky 모드 (문자열의 특정 위치에서 매치 시작)

3. operators

- `^`: 문자열의 시작을 나타냄
- `$`: 문자열의 끝을 나타냄
- `{n,m}`: n개부터 m개까지 반복
- `*`: 0개 이상 반복 (즉, {0,}과 동일)
- `+`: 1개 이상 반복 (즉, {1,}과 동일)
- `?`: 0개 또는 1개 (즉, {0,1}과 동일)
- `()`: 그룹화, 묶음
- `[]`: 문자 클래스, 범위 묶음
- `[^x]`: x를 제외한 문자 (not)
- `|`: OR 연산자, 둘 중 하나 매치
- `.`: 개행 문자를 제외한 모든 문자 매치
- `\`: 이스케이프 문자, 특수 문자를 일반 문자로 인식하게 함
- `\b`: 단어 경계 (word boundary)
- `\B`: 단어 경계가 아닌 위치 (not word boundary)

4. encodeURI(),decodeURI()

- `encodeURI()`: URI 전체를 인코딩할 때 사용
  - 예: `encodeURI("https://example.com/한글 페이지")` → `"https://example.com/%ED%95%9C%EA%B8%80%20%ED%8E%98%EC%9D%B4%EC%A7%80"`
- `decodeURI()`: 인코딩된 URI 전체를 디코딩할 때 사용
  - 예: `decodeURI("https://example.com/%ED%95%9C%EA%B8%80%20%ED%8E%98%EC%9D%B4%EC%A7%80")` → `"https://example.com/한글 페이지"`

5. encodeURIComponent(),decodeURIComponent()

- `encodeURIComponent()`: URI 구성 요소(쿼리 파라미터 등)를 인코딩할 때 사용
  - 예: `encodeURIComponent("한글 페이지")` → `"%ED%95%9C%EA%B8%80%20%ED%8E%98%EC%9D%B4%EC%A7%80"`
- `decodeURIComponent()`: 인코딩된 URI 구성 요소를 디코딩할 때 사용
  - 예: `decodeURIComponent("%ED%95%9C%EA%B8%80%20%ED%8E%98%EC%9D%B4%EC%A7%80")` → `"한글 페이지"`

6. replace, replaceAll

- `replace()`: 문자열에서 첫 번째 매치된 부분만 교체
  - 예: `"hello world".replace("o", "O")` → `"hellO world"`
- `replaceAll()`: 문자열에서 모든 매치된 부분을 교체
  - 예: `"hello world".replaceAll("o", "O")` → `"hellO wOrld"`
  - 참고: `replaceAll()`은 ES2021(ES12)에서 도입되었으므로, 이전 버전에서는 사용할 수 없다.
- 정규식을 써야하는 이유
- 예: 모든 공백 문자를 밑줄(\_)로 교체

```js
const str = "Hello World! This is a test.";
const result = str.replace(/\s/g, "_"); // 정규식 사용
console.log(result); // "Hello_World!_This_is_a_test."
```

7. 정규식 search, match, test

- `search()`: 문자열에서 정규식과 매치되는 첫 번째 위치의 인덱스를 반환. 매치가 없으면 -1을 반환.
  - 예: `"hello world".search(/o/)` → `4`
- `match()`: 문자열에서 정규식과 매치되는 모든 부분을 배열로 반환. 매치가 없으면 null을 반환.
  - 예: `"hello world".match(/o/g)` → `["o", "o"]`
- `test()`: 정규식이 문자열과 매치되는지 여부를 불리언 값으로 반환.
  - 예: `/o/.test("hello world")` → `true`
