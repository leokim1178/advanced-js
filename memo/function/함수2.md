

bind에 대해서 알아보자

function object에는 [[BoundFunctionObject]]라는 내부 슬롯이 있었다
이 내부 슬롯은 함수가 bind 메서드로 생성되었는지를 나타낸다
추가로 [[BoundThis]], [[BoundArguments]]라는 내부 슬롯도 존재한다


```js
fn.call(thisBindingObject, args1, args2, …);  // call
fn.apply(thisBindingObject, [args1, …]);      // apply (array)
const bindFn = fn.bind(thisBindingObject);    // not call, only bind
bindFn(args1, args2, …);  
fn.bind(thisBindingObject, arg1, arg2, …)();  // BoundArguments로 결정
```
그럼 위에서 thisBindingObject와 args1, args2는 어디에 저장될까
thisBindingObject는 [[BoundThis]]에 
args1, args2는 [[BoundArguments]]에 저장된다

그런데 만약 이렇게 사용하면 어떻게 될까
```js
fn.bind(thisBindingObject, arg1, arg2, …)(x1,x2,…);
```
이 경우 arg1, arg2는 [[BoundArguments]]에 저장되고
x1, x2는 이 함수를 실행할때 일회성으로 전달되는 인자가 될것이다
이렇게 bind된 함수는 함수를 부를때마다 BoundArguments가 따라붙게 된다고 보면 된다
